# Experience Principles

## 1. The Arcade Comes First

The main screen should feel like a game room, not a dashboard.

Numbers are allowed, but they should be embedded in playful objects:

- coin counter
- token meter
- cabinet level plate
- prize wall slots
- capsule reveal card

## 2. Every Sync Needs A Payoff

Syncing should not feel like refreshing analytics.

It should feel like collecting rewards:

- lights flicker on
- coin counter climbs
- cabinets pulse
- capsule machine becomes available

## 3. The User Does Not Need To Read Chats

Do not expose full conversation logs in the core experience.

The product exists because the user wants a project-level game layer, not another transcript viewer.

## 4. Token Spend Is Enough

The product should not ask:

- Was this productive?
- Did tests pass?
- Did code ship?
- Was the AI useful?

The product asks only:

- How many tokens were consumed?
- Which project consumed them?
- What can the user unlock now?

## 5. Rewards Should Be Cosmetic And Collectible

The best rewards are visible:

- a brighter cabinet
- a badge on the wall
- a new room theme
- a rare nameplate
- a trophy card

Avoid complex stats that create optimization anxiety.

## 6. The First Minute Must Work

If real token data is unavailable, show a demo arcade with mock projects and mock rewards.

The user should understand the fantasy before configuring anything.

## 7. Keep It Cozy

Visual direction:

- pixel art
- warm neon
- dark arcade room
- gold coins
- chunky UI
- short satisfying animations

Avoid:

- corporate analytics styling
- dense tables
- generic SaaS cards
- combat fantasy
- guilt about cost
